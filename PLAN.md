This repository started as a time tracking tool only. it has features like tracking time for topics, tracking breaks and having simple statistics. recently i also added a simple web daemon to have a gui. I want to migrate this app to a full time management application for which i need to do some major changes. I want you to generate me a multi step plan to orchestrate agents to do these. 

Features we currently have: 
 - local sqlite3 db 
 - web view with calendar and sql 
 - simple cli statistics 

Features I want: 

1. Users should be able to track times for topics and subtopics. e.g. I am doing my maths portion of the day. now im just doing pomo start math. but maybe i want to see in the future which topics took the longest during the semester for my exam preparation. thus i want to do something like pomo start math discrete probability. then the top domain would be math and the sub domain would me discrete probability. we need to also be clear on how to do this, e.g. what if the top domain is multiple words. This should also be possible in the web app as well as the cli
2. the tracked events are basically what has been done for the day. we also want to implement scheduling and planned tasks. this means we need some kind of 2 layers. one layer of planned events and one layer of done events. done would be those tracked, planned the ones generated by the scheduler. 
3. we need to be able to also add single events. the cli and web app should basically be ones calender. we need to be able to add single events to the calender. this should be available in the cli and web app. 
4. we need to be able to add recurring tasks, e.g. going for a run everyday etc. these should be somehow managed by a event type for the scheduler, so that the scheduler can only incorporate events instead of multiple data structures. this should also be able via cli and webapp. 
5. we need to be able to add weekly/monthly/daily tasks which take x amount of time. e.g. my maths class has 5 credits this means i need to do 8h a week for math. this should be linked with the recurring events which can also store topics, e.g. i have a lecture of 1.5h in the math topic, this means i need to do 6.5h math this week. this needs to be splitted into events where i need to study math throughout the week. Also e.g. gym. so we want to go to the gym 4 times a week for 2h. 
6. we need to set some kind of values, e.g. a lunch break around noon. a dinner break around dinner. how long these breaks should be. also when we start our day, e.g. 6,7,8am and when we finish e.g. 10,11,12pm etc. basically when we are ready to actually work/learn. this can also be done via cli and webapp. 
7. the web app is a simple webapp for now with a daemon running. i want the app to look better so we need some ui libraries we can use. if the app isnt lightweight anymore we need to find a fix. either spin up the server everytime or some kind of standby and then the app starts running. 
8. all cli tools that make sense as a tui like we did in delete should be done via tui. 
9. we should also have a dashboard of e.g. how accurately we are following the plan, e.g. doing the stuff exactly on time, and also like how we did, e.g. did we do everything we want to do. 
10. we also need some kind of logic to the following problem. we may have 2 dependant events. e.g. a lecture and a tutorial. in general i want to go to the tutorial after i went to the lecture. 
11. we also need to incorporate breaks after working sessions. e.g. i want to do 2-3h of work inside a block. in this time i may also take some breaks like the pomo technique. e.g. i work for 25m on math. i take 5m break. i do this again. instead of 50m math i should track like 1h on math. for this there should also be a threshold. e.g. every break between 2 sessions of the same top domain which are less than 10, should count to this topic. 

Go through the plan, generate a memory file for codex and also ask if something is unclear. what we also should do is enhance the help menu of pomo because some stuff, e.g. pomo set is really unclear. do you also have any suggestions on tools? Plan this in detail step by step and how it makes sense chronologically feature wise. for every part i will generate a seperate feature branch which then will be done step by step.
